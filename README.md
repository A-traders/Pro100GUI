# Pro100GUI

GUI-оркестратор для пайплайна Pro100 поверх MetaTrader 5 Strategy Tester.

Заменяет ручную последовательность скриптов (`opt_pro100`, `mm_sweep`,
`build_pro100_pdf`) на единое десктоп-приложение: один экран конфигурации,
вкладка с живым прогрессом, JSON-сохранение состояния (с возобновлением
после сбоя), просмотр готовых PDF-результатов.

## Документация для пользователя

Полное руководство по установке и работе с программой:
[**docs/UserGuide.pdf**](docs/UserGuide.pdf).

## Что нужно

- Windows 10 / 11.
- Python 3.11 или новее.
- MetaTrader 5 (любая сборка, например RoboForex MT5 Terminal) -- скачивается отдельно.
- Файл советника `.ex5` -- скачивается вручную из публичного Telegram-поста <https://t.me/xauruspro/16>.

## Быстрый старт (для тех, кто знаком с командной строкой)

```
git clone https://github.com/A-traders/Pro100GUI.git
cd Pro100GUI
pythonw Pro100GUI.pyw
```

При первом запуске `bootstrap.py` сам поставит зависимости
(`PySide6`, `reportlab`, `pypdfium2`, `pypdf`, `Pillow`, `requests`).

После открытия окна -- вкладка **Настройки** -> указать путь к MT5
и к скачанному `.ex5` -> сохранить. Дальше вкладка **Конфигурация**
-> кнопка **Start**.

## Архитектура (5 слоев)

1. **core** -- чистая логика без GUI и MT5: модели конфигурации,
   расчет периодов, фильтры, TWR-математика, кодек pro100 csv.
2. **adapters** -- внешние системы: `TerminalRunner` (запуск
   `terminal64.exe`), `SetFileBuilder`, `IniFileBuilder`,
   `FilesStaging`, `EAVersionChecker` (проверка имени файла советника
   против Telegram-поста), `EARegistry`, `AddFrConfig` (профили AddFr
   для EA _009), `PdfRenderer`, `PdfQC`.
3. **orchestrator** -- DAG задач, состояние, JSON-persistence,
   `EventBus` для асинхронных обновлений GUI.
4. **gui** -- PySide6, 4 экрана: Конфигурация / Прогон / Результаты /
   Настройки.
5. **app** -- shell, логирование, точка входа `Pro100GUI.pyw`.

## Источник советника

Файл `.ex5` советника **не хранится** в репозитории (для этого
включено три рубежа защиты: `.gitignore`, pre-commit hook,
GitHub Actions workflow `no-ea-files`).

Пользователь скачивает советник вручную из публичного канала
<https://t.me/xauruspro/16> и указывает путь к локальному файлу в
настройках. При запуске Pro100GUI читает Telegram-пост и сверяет
имя файла с локальным -- если расходятся, выводит предупреждение
(но не блокирует прогон).
